# BACKUP 2026-06-05

## Version
- Release target: `v0.4.96`
- Main change: WeChat group-only backup and import.

## What To Back Up Before Release
- Keep a copy of the existing SQLite database and `data/wechat-media` before testing real imports.
- Keep existing `.env` and local `config.json` out of any exported group backup.
- Do not commit generated `.json` backup files; they can contain private group chat content.

## New Backup Scope
- Included: group chat activity, local group messages, group memory items, member names, member identity mappings, member identity aliases, and image/media analysis rows.
- Excluded: LLM channels, Skill/multimodal channels, knowledge base, global memories, config, API keys, tokens, embedding vectors, and Honcho sync IDs.
- Import creates ledger tables `wechat_group_backup_imports` and `wechat_group_backup_import_rows` for traceability.

## Safety Notes
- Import requires the current Wechaty account to be online and to return a fresh `bot.Room.findAll()` result.
- Missing groups, duplicate group names, stale room lists, and unconfirmed unique-name matches are skipped.
- Imports merge and deduplicate by group/message/member/media keys; they do not clear or overwrite a target group by default.
- Full media mode may produce large backup files. Media files over 20MB keep metadata but skip file body export.
